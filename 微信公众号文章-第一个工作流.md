# 还在手动复制网页数据？你只需要说一句话，剩下的交给 AI

### 那个熟悉的场景

打开网页 → 选中文字 → 复制 → 粘贴到 Excel → 翻页 → 再复制 → 再粘贴……

一个小说排行榜，几十本书，书名、作者、类型、封面图，光复制粘贴就能耗掉你半小时。更别提数据量大的时候，眼睛盯到酸、手点到麻。

你肯定想过："能不能让工具帮我自动抓？"

然后你去搜了一堆 RPA 教程，看到「CSS 选择器」「XPath」「DOM 树」这些词，默默关掉了页面。

**"我就想扒个数据而已，为什么还得先学前端？"**

### 传统 RPA 的隐形门槛

市面上的自动化工具不少，但大多数都有一个共同问题：**你必须先学会元素定位。**

打开 F12 开发者工具，在一堆 `<div>`、`<span>`、`<class>` 里找到目标元素，写出正确的选择器表达式——这一步就劝退了 90% 的普通用户。

不是说选择器没用。而是说：**工具应该适应人，而不是让人适应工具。**

### 说人话，就能采集数据

九头虫 RPA 做了一件事：**把元素定位这件事，交给了 AI。**

你不用写一行 CSS 选择器，不用按 F12，不用理解什么是 DOM。你要做的，就是用大白话描述你要什么。

比如你想从小说排行榜里采集数据，只需要告诉 AI：

> **帮我们找出「小说排行榜」中「每一项小说卡片」的所有元素根节点**

一句话，AI 自动帮你定位所有小说卡片。

然后继续告诉它你要哪些字段：

| 你想要什么 | 你说的 |
| :--- | :--- |
| 小说名称 | 获取「小说名称」 |
| 小说作者 | 获取「小说作者」 |
| 小说类型 | 获取「小说类型」，它位于「小说作者」下方 |
| 小说封面图 | 获取「小说封面图」 |

每输入一句，点一下发送按钮，AI 就自动把元素找到了。不确定对不对？点预览看一眼，高亮区域一目了然。

**全程没有出现一个 `#`、一个 `.`、一行代码。**

### 右键菜单，即开即用

更妙的是触发方式。

你不需要每次都打开插件面板点「执行」。把流程配成**右键菜单触发**——在目标网页上点一下右键，选对应菜单项，流程立刻跑起来。

整个工作流就两步：**右键菜单 → 数据采集**。连起来，保存，完事。

### 以前 vs 现在

| | 以前 | 现在 |
| :--- | :--- | :--- |
| 定位元素 | 打开 F12，写 CSS 选择器 | 说人话，AI 自动找 |
| 上手门槛 | 得懂前端基础 | 会用浏览器就行 |
| 操作步骤 | 手动复制粘贴，反复切换窗口 | 右键一点，自动出结果 |
| 出错排查 | 盯着选择器一句句调试 | 点预览，高亮对不对一目了然 |
