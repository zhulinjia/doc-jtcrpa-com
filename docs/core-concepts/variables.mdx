# 变量与表达式

九头虫 RPA 使用 `{{ }}` 双花括号语法引用变量和执行表达式。所有节点参数中均可用此语法动态获取上下文数据。

## 变量基础

上下文变量在流程运行期间全局可见。通过「[设置变量](/components/general-components/set-variable)」节点声明，或在某些节点（[数据采集](/components/general-components/data-collection)、[等待 HTTP](/components/network-components/wait-http) 等）中自动产出。引用变量时使用双花括号包裹：

```
{{变量名}}
```

### 在文本中嵌入变量

`{{ }}` 支持两种模式：

| 模式 | 示例 | 说明 |
|------|------|------|
| 纯表达式 | `{{score}}` | 整个字符串是一个表达式，保留原始类型 |
| 混合文本 | `共 {{count}} 条记录` | 变量嵌入文本，结果为字符串 |

```
纯表达式：{{username}}        → Alice（字符串）
纯表达式：{{score + 5}}        → 100（数字）
混合文本：Score: {{score}}/100 → "Score: 95/100"（字符串）
```

---

## 变量作用域

| 作用域 | 声明方式 | 可见范围 |
|--------|----------|----------|
| 全局变量 | 设置变量节点（默认） | 当前流程所有节点 |
| 局部变量 | 设置变量节点（局部模式） | 仅当前分支内可见 |
| 节点自动产出 | 数据采集、等待 HTTP 等 | 从产出位置起全局可见 |

---

## 支持的操作

变量表达式底层基于 [jsep](https://github.com/EricSmekens/jsep) 解析引擎，支持完整的 JavaScript 表达式语义。

### 算术运算

```
{{price * quantity}}
{{total / 100}}
{{score % 10}}
{{base ** 2}}
```

### 比较运算

```
{{score >= 60}}
{{status === 'ok'}}
{{count !== 0}}
```

### 逻辑运算

```
{{isActive && count > 5}}
{{isExpired || isCancelled}}
{{!isEmpty}}
```

### 属性访问

支持点号和方括号访问对象属性及数组元素：

```
{{user.name}}
{{user.profile.city}}
{{list[0]}}
{{list[index]}}
```

### 三元运算

```
{{score >= 60 ? '及格' : '不及格'}}
```

## 内置函数

以下函数可在表达式中直接调用，无需声明。

| 函数 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `getPageUrl()` | 无 | string | 当前页面完整 URL |
| `getUUID()` | 无 | string | 生成 UUID v4 |
| `getTimeStamp()` | 无 | number | 当前时间戳（毫秒） |
| `random(min, max)` | number, number | number | 生成 [min, max] 范围内的随机整数 |

**示例**：

```
{{getPageUrl()}}
{{getUUID()}}
{{random(1000, 9999)}} 
```

## 常见问题

### 变量引用了但显示为空

**现象**：在节点参数中写了 `{{变量名}}`，但运行时替换为空。

**原因**：变量未在引用前声明，或作用域不匹配（在分支外部引用了分支内部的局部变量）。

**解决**：
1. 确认「[设置变量](/components/general-components/set-variable)」节点在流程顺序上位于引用之前；
2. 使用「[打印输出](/components/general-components/print-output)」节点在引用前打印变量值，确认变量存在且类型正确。

### 在 JSON 中使用变量

**现象**：在 JSON 配置字段中写了变量表达式，但运行时报错或结果不符合预期。

**原因**：变量在 JSON 中的解析与正常 JSON 类型声明一致——变量解析成什么类型，最终值就是什么类型。字符串变量必须加引号，否则会产生不合法 JSON。

**解决**：按 JSON 的正常写法使用变量：

```json
// ✅ 字符串变量需要引号
{"name": "{{username}}"}        → {"name": "Alice"}

// ✅ 数字变量不需要引号
{"count": {{total}}}            → {"count": 5}

// ✅ 布尔值不需要引号
{"active": {{isActive}}}        → {"active": true}

// ❌ 字符串变量不加引号 → 不合法 JSON
{"name": {{username}}}          → {"name": Alice}
```

### 表达式计算结果与预期类型不符

**现象**：`{{count + 1}}` 结果为字符串 `"11"` 而非数字 `2`。

**原因**：变量 `count` 的实际值是字符串 `"1"`，`+` 运算执行了字符串拼接。

**解决**：使用「[设置变量](/components/general-components/set-variable)」节点时选择正确的变量类型（数字而非字符串），确保上游节点产出的变量类型符合预期。
