# 快速上手

九头虫 RPA 是一款**浏览器自动化插件**，让你无需写代码就能自动完成网页操作——填表、采集数据、批量处理，统统交给它。

**为什么选九头虫 RPA？**

- **零基础也能用**：不需要懂 CSS 选择器，也不需要会编程。内置的 AI 自然语言引擎让你用大白话描述想要什么元素，AI 自动帮你定位。
- **开箱即用**：拖拽节点、连连线、点执行，三步搭出一个自动化流程。就像搭积木一样简单。
- **浏览器原生体验**：作为浏览器扩展，直接运行在你正在操作的网页上，无需安装额外软件，也不受跨域限制。

本页带你完成安装和第一个工作流，大约需要 **5 分钟**。

---

## 1. 安装插件

### 从应用商店安装（推荐）

可以直接从浏览器官方商店搜索安装：

- **Chrome 浏览器**：前往 [Chrome 应用商店](https://chromewebstore.google.com/detail/dopdcakmheonlfplanpbeodjicmpalkm?utm_source=item-share-cb) 搜索「九头虫 RPA」
- **Edge 浏览器**：前往 [Edge 加载项](https://microsoftedge.microsoft.com/addons/detail/pbojbhbhgenkoebpdcpojfnohjdjckjb) 搜索「九头虫 RPA」

:::tip[版本更新]
应用商店版本会自动更新，推荐大多数用户选择此方式。
:::

### 手动安装（离线包）

如果无法访问应用商店，可从百度网盘下载离线包手动安装：

> 下载链接：[百度网盘 - 九头虫 RPA](https://pan.baidu.com/s/1ipAYJ-A7GVtqB6KYzH381g?pwd=ew24)（请替换为实际链接）

下载得到 `九头虫RPA.zip` 压缩包后，**推荐直接拖拽安装**（最简单）：

1. 打开浏览器，地址栏输入 `chrome://extensions`（Edge 浏览器输入 `edge://extensions`）
2. 打开页面右上角的 **「开发者模式」** 开关
3. 将下载的 `九头虫RPA.zip` **直接拖入**浏览器扩展管理页面，松手即完成安装

> 这种方式不需要解压，浏览器会自动识别并加载。

**备选方式**：如果拖拽不生效，也可以解压后手动加载：

1. 将 ZIP 文件解压到本地目录（建议放在独立文件夹，不要放桌面或临时目录）
2. 打开 `chrome://extensions`（Edge 输入 `edge://extensions`），开启「开发者模式」
3. 点击 **「加载已解压的扩展程序」**，选择解压出的文件夹

安装完成后，浏览器工具栏会出现九头虫 RPA 图标。如果没有，点击工具栏右侧的拼图图标，找到九头虫 RPA，点击图钉固定。

:::caution[注意]
无论哪种方式，插件文件夹都**不要删除或移动位置**，否则浏览器会找不到插件文件。
:::

### 示例：打开网页并采集数据

**步骤一：添加触发节点**

从左侧节点库拖入「[手动执行](/components/triggers/manual-execute)」触发器到画布，作为流程入口。

**步骤二：打开目标网页**

拖入「[打开网页](/components/browser-control/open-page)」节点（位于「浏览器控制」分类），连接到触发器。在配置面板中输入目标 URL，例如：

```
https://www.example.com
```

**步骤三：等待页面加载**

拖入「[等待元素](/components/general-components/wait-element)」节点（位于「通用组件」分类），在「目标元素」输入框中点击 🤖 切到 [AI 模式](/core-concepts/css-selectors#ai-自然语言选择器推荐新手使用)，描述你等待出现的元素即可：

```
页面标题
```

> 也可以直接写 CSS 选择器，比如 `h1`。但对于新手，AI 模式更友好——你不需要懂任何 CSS 语法。

**步骤四：采集页面数据**

拖入「[数据采集](/components/general-components/data-collection)」节点（位于「通用组件」分类）。同样使用 [AI 模式](/core-concepts/css-selectors#ai-自然语言选择器推荐新手使用)：用大白话描述你要提取的行和列，AI 自动帮你定位。

例如你想采集一个商品列表，行描述写 `所有商品卡片`，列描述写 `商品名称`、`价格`、`销量` 即可，无需手写选择器。

**步骤五：输出结果**

拖入「[输出打印](/components/general-components/print-output)」节点（位于「通用组件」分类），连接到数据采集节点。它会把采集到的数据输出到日志面板。

**步骤六：运行流程**

连接所有节点后，点击顶部工具栏的「执行」按钮，流程随即开始运行。在下方日志面板查看实时执行状态和采集结果。

> 完整流程：**手动执行 → 打开网页 → 等待元素 → 数据采集 → 输出打印**

## 3. 基础概念

使用九头虫 RPA 前，建议先了解三个贯穿所有节点的核心概念：

- **[元素选择器](/core-concepts/css-selectors#ai-自然语言选择器推荐新手使用)**：定位页面元素。**推荐新手直接用 [AI 自然语言模式](/core-concepts/css-selectors#ai-自然语言选择器推荐新手使用)**——输入大白话描述即可，零基础也能上手；熟悉后可进阶使用 CSS 选择器及 `:contains`、`:shadow`、`:col` 等自定义伪类
- **[变量与表达式](/core-concepts/variables)**：`{{ }}` 双花括号语法，支持算术、比较、逻辑运算及内置函数
- **[数据清理管道](/core-concepts/data-transform)**：33 个内置操作符，对采集数据进行清洗、转换和格式化

另外两个基础概念：

- **节点**：流程的最小执行单元，每个节点完成一个具体动作（如点击、输入、等待）。
- **连线**：节点之间的箭头，决定执行顺序和数据流向。

## 常见问题

### 插件安装后工具栏没有图标

**现象**：浏览器扩展管理页面显示已安装，但工具栏不显示图标。

**原因**：浏览器默认将新安装的扩展折叠到扩展菜单中。

**解决**：点击工具栏右侧的拼图图标（扩展菜单），找到九头虫 RPA，点击图钉固定到工具栏。

### 手动安装的 ZIP 包无法加载

**现象**：加载解压文件夹后提示"清单文件缺失"或"文件损坏"。

**原因**：可能是解压不完整，或者选错了文件夹层级。

**解决**：
1. 确认 ZIP 包下载完整（对比网盘标注的文件大小）
2. 重新解压，选择"解压到同名文件夹"
3. 加载时**选择解压后包含 `manifest.json` 的那一级文件夹**，不要选它的上级或下级目录

### 不知道用哪种方式触发流程

**现象**：编辑好流程后，不清楚如何启动它。

**原因**：九头虫 RPA 提供了多种触发器，各自适用于不同场景。

**解决**：根据使用场景选择最合适的触发器：

- **手动执行**：点击画布顶部的「执行」按钮直接运行，适合测试和临时任务。
- **右键菜单**：在网页上选中目标区域后右键触发，适合需要快速对特定元素执行操作的场景。
- **快捷键触发**：绑定组合键一键启动，适合高频重复操作。
- **定时触发**：按 Cron 表达式周期执行，适合定时巡检、数据同步等无人值守场景。
- **元素监控 / 网络拦截 / URL 匹配**：当页面出现指定元素、接口返回数据或 URL 变化时自动执行，适合事件驱动型监听场景。

如果只是试用或测试流程，直接选「手动执行」。

### 流程设置了被动触发但一直不启动

**现象**：使用元素监控、网络拦截或 URL 匹配等被动触发器，流程始终没有自动执行。

**原因**：被动触发器需要满足特定条件才会激活，常见原因有以下几种。

**解决**：
1. 检查触发器配置中的选择器或 URL 规则是否正确，先在浏览器开发者工具中验证
2. 确认目标事件（DOM 节点出现、网络请求、URL 变化）确实在页面上发生——部分单页应用的路由切换可能不触发 URL 匹配
3. 元素监控类触发器需要目标节点已存在于 DOM 中（或等待其出现），如果页面动态渲染，适当增大监控间隔或延长超时时间
4. 查看插件日志面板
