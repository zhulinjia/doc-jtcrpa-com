# 九头虫 RPA

**浏览器原生、零代码的自动化插件。** 不用写脚本，不用学框架，装好就能跑 —— 从日常重复操作到无人值守数据采集，一个插件搞定。

![九头虫 RPA](/img/ui.png)

---

## 你需要会什么

九头虫 RPA 面向的是**没有编程背景的普通用户**。你不需要会写代码，不需要懂框架。但要顺利使用基础核心功能（打开网页、点击按钮、填写表单、采集数据），你至少需要：

- **会用浏览器** — 日常使用 Chrome 或 Edge 上网，知道地址栏、标签页、工具栏在哪
- **能安装浏览器扩展** — 知道怎么从应用商店或离线包安装插件
- **分得清网页上的基本元素** — 知道什么是按钮、输入框、下拉菜单、链接、图片
- **有"按步骤做事"的流程思维** — 能把自己的操作拆成"先打开A → 等B出现 → 点C → 填D"这样的步骤链

**如果你满足以上四点，就可以开始用了。** CSS 选择器、变量、表达式等概念文档里都会教，不需要提前懂。

---

## 你遇到的问题

做浏览器自动化的时候，你是不是也踩过这些坑：

- **写脚本太累。** 打开页面 → 等加载 → 找元素 → 处理弹窗 → 写判断逻辑 → 处理异常……一个简单流程调试半天。
- **元素定位不准。** 脚本写好了，隔天页面改了个 class 就挂。iframe 嵌套深了更是噩梦，一层层 `switch_to_frame` 写到怀疑人生。
- **出错必须人盯着。** 网络波动、弹窗拦截、元素还没渲染——任何一个意外都能让脚本卡死。想加个重试？又得多写几十行。
- **批量跑的时候打架。** 同时开几个实例处理任务，资源抢占、状态串了，排查起来一头包。

## 九头虫怎么解决

### 拖拽编排，告别脚本

在画布上拖节点、连箭头，流程逻辑一目了然。条件分支、循环、子流程嵌套、多路并行——你画成什么样就跑成什么样。**改流程就是改图，不用翻代码。**

### 元素定位稳得住

内置多模态查询引擎，支持 CSS 选择器扩展（`:contains`、`:shadow`、`:col` 等 8 个自定义伪类），能穿透多层 iframe 精准命中目标元素。即使页面动态渲染、DOM 延迟加载，也能等它出现再执行。

### 出错不用人救

每个节点独立配置重试策略：重试几次、间隔多久、全失败了怎么办（停止 / 跳过继续 / 跳出循环 / 跳过本轮）。**写一次配置，从此放心跑无人值守。**

### 并发可控，不打架

每个触发器独立限制最大并发数，超出的请求直接丢弃。短时间重复触发自动去重，**不会因为误操作或网络抖动把同一个流程跑出七八个实例。**

![并发可控](/img/config.png)

### 不只是模拟点击

WebSocket 长连接、SSE 流式指令、网络层拦截 XHR/Fetch —— 直接绕过前端 UI 从底层拿数据。定时任务、右键菜单、快捷键、DOM 监控、URL 匹配……**十种触发方式覆盖一切启动场景。**

## 模块全景

| 模块 | 能力 |
|------|------|
| **触发器** | 手动执行、右键菜单、快捷键、定时任务、DOM 监控、网络拦截、URL 匹配、WebSocket、SSE、信号触发 |
| **通用组件** | 变量管理、数据采集、异步等待、执行沙箱代码、文件读取、控制台输出 |
| **流程控制** | 条件/多路分支、循环、子流程调用、汇聚、延迟、返回、信号收发 |
| **浏览器控制** | 窗口/标签页管理、导航控制、iframe 切换、弹窗处理 |
| **事件交互** | 点击、悬浮、按键、滚动、文字输入（模拟打字）、表单提交、文件上传/下载、剪贴板 |
| **网络组件** | HTTP 请求拦截与发送、响应数据结构化提取 |
