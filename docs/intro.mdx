# 九头虫 RPA

**会打字就能用的浏览器自动化插件。** 不用写代码，不用学 [CSS 选择器](/core-concepts/css-selectors)——用大白话告诉 AI 你要操作什么，它帮你搞定。

![九头虫 RPA](/img/ui.png)

---

## 跟其他 RPA 工具有什么不一样

大多数 RPA 工具要求你**先学会元素定位**才能开始用。你得打开开发者工具、看懂 DOM 树、写出正确的 [CSS 选择器](/core-concepts/css-selectors) 或 XPath——光这一步就劝退了大量普通用户。

九头虫 RPA 把这件事做成了**你用大白话描述，AI 自动寻址**：

- 想点按钮？写 `点击登录按钮`
- 想填输入框？写 `密码输入框`
- 想从表格里找某一行？写 `点击订单号 001 那行的查看详情`

不需要打开 F12，不需要理解什么是 class、什么是 id。**你描述页面上的元素，就像你跟同事说话一样。** 当然，如果你熟悉 [CSS 选择器](/core-concepts/css-selectors)，也可以直接手写——两种模式随时切换。

---

## 你需要会什么

- **会用浏览器** — 日常用 Chrome 或 Edge 上网
- **能安装浏览器扩展** — 从应用商店点一下「安装」
- **有"按步骤做事"的思路** — 能把操作拆成"先打开A → 等B出现 → 点C → 填D"

满足以上三点就可以开始了。[CSS 选择器](/core-concepts/css-selectors)、[变量与表达式](/core-concepts/variables)、[数据清理管道](/core-concepts/data-transform) 都是可选的进阶能力，不学也能完成大部分自动化需求。

## 能做什么

| 场景 | 怎么做 |
|------|--------|
| 登录网站 | 拖「[打开网页](/components/browser-control/open-page)」+「[输入文字](/components/event-interaction/text-input)」×2 +「[点击事件](/components/event-interaction/click)」，每个节点用 AI 描述目标 |
| 采集数据 | 拖「[数据采集](/components/general-components/data-collection)」，AI 模式下描述行和列，数据自动导出 |
| 批量填表 | 拿 Excel 数据源，循环填入网页表单 |
| 定时巡检 | 设一个[定时触发器](/components/triggers/scheduled-trigger)，每天自动检查页面状态并截图 |
| 监听新消息 | 配一个[元素监控触发器](/components/triggers/element-monitor)，页面出现新消息立刻采集 |

## 不止一种启动方式

大多数自动化工具只有一种启动方式：点「运行」按钮。九头虫 RPA 把启动方式分成了两类，适配不同的使用场景：

### 主动触发——你说了算

需要你手动操作一下才启动，适合**人在电脑前、随时可介入**的场景：

| 方式 | 适合场景 |
|------|----------|
| [手动执行](/components/triggers/manual-execute) | 在画布里点「执行」就跑，测试和临时任务最常用 |
| [右键菜单](/components/triggers/context-menu) | 在网页上选中内容 → 右键 → 点菜单项触发 |
| [快捷键](/components/triggers/keyboard-shortcut) | 绑一组组合键，一键启动，高频操作首选 |

### 被动触发——到点自动跑

不需要人，条件满足就自动启动，适合**无人值守、长期运行**的场景：

| 方式 | 适合场景 |
|------|----------|
| [定时任务](/components/triggers/scheduled-trigger) | 按 Cron 表达式周期执行，比如每天 9 点巡检、每小时同步一次数据 |
| [元素监控](/components/triggers/element-monitor) | 页面 DOM 出现指定元素时自动触发，比如「新消息到了」「弹窗出来了」 |
| [网络拦截](/components/triggers/network-intercept) | 拦截页面发出的 XHR/Fetch 请求，匹配到目标接口就触发 |
| [URL 匹配](/components/triggers/url-match) | 监听到浏览器地址变化时触发，适配 SPA 单页应用的路由跳转 |
| [WebSocket](/components/triggers/websocket-trigger) / [SSE](/components/triggers/sse-trigger) | 接收服务端实时推送的指令，远程驱动浏览器执行流程 |
| [信号触发](/components/triggers/signal-trigger) | 一个流程跑完后发信号，另一个流程收信号启动——多流程联动 |

> **跟别的 RPA 工具拉开差距的，是这 6 种被动触发方式。** 它们让九头虫 RPA 从「操作辅助工具」变成了「7×24 自动值守系统」——配置好之后就不用管了，条件满足自动跑。

## 出错不用人盯着

每个节点独立配置重试策略：重试几次、间隔多久、全失败了怎么办（停止 / 跳过继续 / 跳出循环 / 跳过本轮）。网络波动、弹窗拦截、元素延迟渲染——这些常见意外都能自动兜底，**配一次，放心跑无人值守。**
