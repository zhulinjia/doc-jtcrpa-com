# 创建第一个工作流

本页带你从零开始搭建一个简单的数据采集流程——右键菜单触发、AI 自然语言采集数据。全程不需要写一行 CSS 选择器。

打开插件主面板，点击「新建流程」进入画布编辑界面，开始搭建。

## 示例：右键菜单触发数据采集

### 步骤一：添加触发节点

从左侧节点库拖入「[右键菜单](/components/triggers/context-menu)」触发器到画布，作为流程入口。

### 步骤二：添加数据采集节点

拖入「[数据采集](/components/general-components/data-collection)」节点（位于「通用组件」分类），连接到右键菜单触发器。

在配置面板中点击 🤖 切换到 [AI 自然语言模式](/core-concepts/css-selectors#ai-自然语言选择器推荐新手使用)，用大白话描述你要提取的行和列，让 AI 帮你定位元素。

本示例以「小说排行榜」页面为例，采集排行榜中的小说信息。

#### 获取行

在「行描述」输入框中写入：

```
帮我们找出【小说排行榜】中【每一项小说卡片】的所有元素根节点
```

输入完成后，点击输入框左侧的 **发送按钮**，AI 才会开始解析你的描述并定位元素。AI 寻址完成后，可以点击 **预览按钮** 确认高亮的行元素是否覆盖了所有小说卡片，确保范围正确。

#### 获取列

依次添加以下列，每写一列都记得点击发送按钮触发 AI 寻址：

| 列名 | 描述 |
| :--- | :--- |
| 小说名称 | `获取【小说名称】` |
| 小说作者 | `获取【小说作者】` |
| 小说类型 | `获取【小说类型】，它位于【小说作者】下方` |
| 小说封面图 | `获取【小说封面图】` |

> 对于「小说封面图」这一列，需要在「输出属性」中将默认的「文本内容提取」改为 **「属性值提取」**，属性名填写 `src`，这样才能拿到图片地址而非空文本。

所有列配置完成后，建议点击预览按钮整体检查一遍，确认每一列取到的内容都符合预期。无需手写任何 CSS 选择器。

### 步骤三：运行流程

连接两个节点（**右键菜单 → 数据采集**）后，点击顶部工具栏的「保存」按钮保存流程。之后在小说排行榜页面上右键即可触发该工作流，在下方日志面板查看采集结果。
