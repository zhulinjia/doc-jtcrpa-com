# 输入文字

输入文字节点向输入框或文本域填写内容，支持模拟逐字输入和输入后的键盘操作。

<a href="/img/输入文字.png" target="_blank"><img src="/img/输入文字.png" alt="输入文字" style={{maxWidth: 480, width: '100%'}} /></a>

## 使用示例

**场景：填写登录表单**

登录页面有用户名和密码两个输入框，填完后按回车提交。

```
输入文字(用户名输入框, 填 admin)
输入文字(密码输入框, 填 123456)
```

第二个输入文字节点开启「模拟逐字输入」并选择换行方式为「回车」，模拟真人打字后敲回车提交。

目标元素支持 [AI 自然语言模式](/core-concepts/css-selectors#ai-自然语言选择器推荐新手使用)——直接写 `用户名输入框`、`密码输入框` 即可，无需手写 CSS 选择器。也支持 `{{变量名}}` 引用动态内容：

```
输入文字(搜索框, 填 {{keyword}})
```

## 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 目标元素 | CSS 选择器 | — | 必填。指向 `<input>` 或 `<textarea>`；指向 `<label>` 时自动找到关联输入框 |
| 文本内容 | 文本 | — | 必填。支持 `{{变量名}}` 引用 |
| 换行方式 | 下拉选择 | 无 | 无 / 回车(Enter) / Shift+回车 / Tab |
| 模拟逐字输入 | 开关 | false | 开启后逐字符输入，模拟真人打字速度 |

## 常见问题

### 输入框显示了文字但表单校验不通过

文字已填入但提交时报校验不通过，通常是前端框架未检测到值变化。

**解决**：开启「模拟逐字输入」；或在输入文字后追加「[触发事件](./trigger-event)」节点，手动派发 `input` 事件。
