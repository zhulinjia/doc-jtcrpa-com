# 悬浮事件

悬浮事件节点将鼠标悬停在指定元素上，触发 hover 相关行为。

## 概述

将鼠标移动到目标元素上并停留。可配合修饰键使用（如按住 Ctrl 再悬浮）。

<a href="/img/悬浮事件.png" target="_blank"><img src="/img/悬浮事件.png" alt="悬浮事件" style={{maxWidth: 480, width: '100%'}} /></a>

## 快速上手

以下示例演示了通过「右键菜单」触发悬浮事件的完整流程：

- **触发器**：[右键菜单](../triggers/context-menu) — 在匹配页面上右键启动工作流
- **动作**：悬浮事件 — 将鼠标悬停在 `#target-hover` 元素上
- **执行页面**：`https://jtcrpa.com/test/index.html`

<a href="/img/悬浮事件示例.png" target="_blank"><img src="/img/悬浮事件示例.png" alt="悬浮事件示例" style={{maxWidth: 480, width: '100%'}} /></a>

执行时，先在匹配页面右键选择菜单项启动流程，随后悬浮事件节点将鼠标移动到目标元素 `#target-hover` 上并停留，触发 hover 相关行为。

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 目标元素 | 文本 | — | [CSS 选择器](../../core-concepts/css-selectors)，必填 |
| 修饰键 | 多选 | — | 多选：`Ctrl`、`Shift`、`Alt`、`Meta` |

## 常见问题

### 悬停后目标元素显示一下就消失了

**现象**：hover 触发的界面弹出后瞬间消失。

**原因**：后续节点立即执行了其他操作导致焦点转移，或页面需要持续悬停才能维持显示。

**解决**：在悬浮节点后紧接操作目标元素的节点（如点击展开的菜单项），避免中间插入其他节点。
