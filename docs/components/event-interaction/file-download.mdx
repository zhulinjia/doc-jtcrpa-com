# 下载文件

下载文件节点支持两种工作模式：传入 URL 直接下载网络文件，或传入文本/JSON 等纯数据由系统自动转为文件下载。

## 概述

节点会根据你填写的「下载 URL」自动判断模式：

- **填的是网址**（`https://` / `http://` / `ftp://` / `file://` 开头）→ 直接下载该网络文件。
- **填的是数据**（变量、JSON 对象、普通文本等）→ 根据「保存文件名」的后缀自动转换格式并触发下载。

「下载 URL」和「保存文件名」均支持 `{{变量名}}` 引用上游数据。

## 快速上手

### 场景一：下载网络文件

1. 拖入「下载文件」节点
2. 「下载 URL」填入文件链接，如 `https://example.com/report.pdf`
3. 「保存文件名」填入 `report.pdf`（可选，不填则用服务端返回的原始文件名）
4. 执行

### 场景二：将数据存为文件

假设上游「网络拦截」节点捕获到一批 JSON 数据，你想把它导出为 Excel：

1. 拖入「下载文件」节点
2. 「下载 URL」填入 `{{responseData}}`——即上游捕获到的数据变量
3. 「保存文件名」填入 `export.xlsx`——后缀决定输出格式
4. 执行，浏览器自动下载 `export.xlsx`

> 如果数据是对象数组，Excel 和 CSV 格式会自动将每个对象的键作为表头。

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 下载 URL | 文本 | — | **必填**。支持网址（直接下载）或数据变量（自动转文件）。支持 `{{变量名}}` |
| 保存文件名 | 文本 | `download` | 可选。支持 `{{变量名}}`。后缀名决定数据转文件的格式，见下方 |

### 数据转文件：后缀与格式对照

当你填的「下载 URL」不是网址时，系统根据「保存文件名」的后缀选择输出格式：

| 后缀 | 输出格式 | 说明 |
|------|----------|------|
| `.txt` | 纯文本 | 对象数组会转为制表符分隔的表格文本 |
| `.json` | JSON | 带缩进的格式化 JSON |
| `.csv` | CSV | UTF-8 编码（含 BOM），对象数组自动生成表头 |
| `.xlsx` / `.xls` | Excel | 对象数组自动转为工作表 |

> 后缀不在此表中的，按 `.txt` 处理。

## 常见问题

### 填了网址但下载的是文件内容而不是文件本身

**现象**：填入 `https://example.com/api/data`，下载下来是 API 返回的 JSON，不是想要的文件。

**原因**：URL 返回的不是二进制文件而是文本数据，浏览器按文件内容下载了。

**解决**：这其实是正确的行为。如果只是想保存 API 返回的数据，确认「保存文件名」后缀正确即可（如 `.json`）。

### 数据转文件时打开了乱码

**现象**：数据存为 `.csv`，用 Excel 打开后中文乱码。

**解决**：下载的 CSV 是 UTF-8 编码。用 Excel 打开时，通过「数据 → 从文本/CSV 导入」并选择 UTF-8 编码即可正确显示。

### 短时间内重复执行只下载了一次

**现象**：循环中连续执行两次下载文件，第二次没有触发下载。

**原因**：节点内置 10 秒冷却期，同一内容的重复下载会被跳过。

**解决**：如果确实需要重复下载，在两次下载之间插入「[延迟](../flow-control/delay)」节点等待超过 10 秒。
