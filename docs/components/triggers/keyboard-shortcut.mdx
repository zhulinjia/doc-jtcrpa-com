# 快捷键触发

绑定一组键盘组合键，在匹配页面上按下即可一键启动工作流。适合高频重复操作，不用切面板、不用找菜单。

<a href="/img/键盘快捷键.png" target="_blank"><img src="/img/键盘快捷键.png" alt="键盘快捷键" style={{maxWidth: 480, width: '100%'}} /></a>

## 配置步骤

1. 拖入「快捷键触发」节点到画布，作为流程起点
2. 在「主按键」下拉中选择一个按键，比如 `A`
3. 在「修饰键」中勾选组合键，比如勾上 `Ctrl` + `Shift`
4. 配置面板会实时预览当前组合键效果：`Ctrl + Shift + A`
5. 可选填写「URL 匹配模式」限定生效页面
6. 保存后，在匹配页面上按下组合键即可启动流程

## 参数说明

### 主按键

支持的按键范围：

```
字母   A - Z
数字   0 - 9
功能   F1 - F12
方向   ↑ ↓ ← → (ArrowUp / ArrowDown / ArrowLeft / ArrowRight)
其他   Enter、Space、Tab、Escape、Backspace
```

### 修饰键

可多选，至少选一个：

```
Ctrl           —— Windows 的 Ctrl / Mac 的 Control
Shift          —— 上档键
Alt            —— Windows 的 Alt / Mac 的 Option
Command / Win  —— Mac 的 Command / Windows 的 Win 键
```

:::tip
建议避开浏览器和系统常用快捷键（如 `Ctrl+S` 保存、`Ctrl+W` 关闭标签页），用三键组合（如 `Ctrl+Shift+A`）可以大幅降低冲突概率。
:::

### URL 匹配模式

限定快捷键在哪些页面生效。**支持多个规则，一行一个**。格式遵循 Chrome [match patterns](https://developer.chrome.com/docs/extensions/develop/concepts/match-patterns)。

```
https://example.com/orders/*
https://*.taobao.com/*
```

不填则**所有页面**都可以用这个快捷键。

## 使用示例

### 示例：同一快捷键在不同页面做不同的事

假设你只有一组习惯的组合键 `Ctrl+Shift+D`，但需要在订单页和商品页上分别做不同的自动化操作。

**流程 A — 在订单页导出订单数据：**

```
快捷键：Ctrl+Shift+D
URL 匹配：
  https://admin.example.com/orders/*

→ 在订单列表页按下 Ctrl+Shift+D，自动导出订单数据
```

**流程 B — 在商品页导出商品数据：**

```
快捷键：Ctrl+Shift+D
URL 匹配：
  https://admin.example.com/products/*

→ 在商品列表页按下 Ctrl+Shift+D，自动导出商品数据
```

同一个 `Ctrl+Shift+D`，靠 URL 匹配区分在哪个页面触发哪个流程。打开百度等无关页面时，两个流程都不会触发。

## 使用场景

快捷键适合**每天要重复很多次**的操作：

- 在客服系统中，`Ctrl+Shift+F` 一键填入常用回复模板
- 在订单页面，`Ctrl+Shift+D` 一键导出当前订单数据
- 在任意页面，`Ctrl+Shift+I` 一键截图并保存

如果需要**完全自动、不需要按键触发**的场景，用「[定时任务](/components/triggers/scheduled-trigger)」「[元素监控](/components/triggers/element-monitor)」等被动触发器。

## 常见问题

### 按下快捷键后没有反应

**现象**：配置了快捷键，但按下后流程不启动。

**原因**：快捷键与浏览器或系统原生快捷键冲突，被优先拦截了。

**解决**：换一个更复杂的组合键（三键甚至四键），比如 `Ctrl+Shift+Alt+K`。也可以在 Chrome 地址栏输入 `chrome://extensions/shortcuts` 查看当前所有扩展的快捷键分配情况。

### 想用的按键在列表里找不到

**现象**：主按键下拉中没有想要的按键。

**解决**：主按键支持手动输入，直接在输入框里敲你要的按键名即可。
