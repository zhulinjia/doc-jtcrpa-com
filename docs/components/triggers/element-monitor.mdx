# 元素监控

元素监控触发器监听指定页面上的 DOM 变化，子元素增减或属性变化时自动触发工作流。

## 概述

基于浏览器的 MutationObserver API，在匹配的页面上监听目标元素的子节点列表变化（`childList`）或子元素属性变化（`attributes`）。变化发生时，工作流自动在匹配页面上启动。

**注意**：元素监控必须搭配 URL 匹配模式使用——先限定监控的页面范围，再指定要监控的 DOM 元素。

## 快速上手

1. 拖入「元素监控」节点到画布
2. 填写「匹配页面」——限定在哪些页面监控（必填），如 `*://example.com/*`
3. 填写「目标元素」——要监控的父元素 CSS 选择器（必填）
4. 选择「监控类型」：子元素增减 或 子元素属性变化
5. 当目标元素下发生匹配类型的变化时，工作流自动启动

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 匹配页面 | 文本 | — | Chrome match patterns 格式，限定监控的页面范围，必填。参见 [URL 匹配模式](../../reference/match-patterns) |
| 目标元素 | 文本 | — | 要监控的父元素 CSS 选择器，必填 |
| 监控类型 | 下拉选择 | — | 必填。`childList` — 子元素增减；`attributes` — 子元素属性变化 |

## 常见问题

### 元素变化了但监控没触发

**现象**：页面上的元素确实在变化，但流程没有启动。

**原因**：匹配页面 URL 模式与实际页面不匹配，或目标元素选择器指向了错误的父元素。

**解决**：
1. 先确认「匹配页面」是否正确
2. 判断目标元素选择器是否正确，点击右下角 👁 预览按钮验证是否能命中目标元素。
