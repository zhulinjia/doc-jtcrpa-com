# 执行代码

执行代码节点用于运行自定义 JavaScript 代码，实现标准节点覆盖不到的扩展逻辑。

## 概述

代码支持 `{{变量名}}` 引用上游变量，引用在代码执行前被替换为实际值。支持 async/await 异步操作。

根据是否开启用户脚本权限，有两种执行环境：

| 执行环境 | 能否操作 DOM | 适用场景 |
|---------|------------|---------|
| **沙箱**（未开启权限） | 不能 | 纯数据计算、JSON 处理、数学运算等 |
| **页面上下文**（已开启权限） | 可以 | DOM 操作、访问页面全局变量、调用页面 API |

未开启用户脚本权限时代码仍然可以正常执行，只是运行在沙箱环境中，无法访问页面的 `document`、`window` 等对象。

## 使用方法

拖入节点后，在代码编辑器中编写 JavaScript 代码。代码中用 `{{变量名}}` 引用上游变量。

### 返回值

代码的 `return` 值即为输出结果。返回值必须是可序列化的 JSON 兼容类型——String、Number、Boolean、Object、Array。不能返回 DOM 元素、函数、Symbol 等不可序列化的值。

### 输出变量

填写变量名后，返回值存入该变量供下游引用。不填则返回值丢弃，仅执行代码的副作用。

### 典型场景

- **纯数据计算**：多字段聚合、嵌套对象转换等内置函数难以表达的逻辑（无需开启权限）
- **DOM 操作**：页面中有大量元素需要按自定义规则处理（需开启权限）
- **调用页面 API**：执行页面中已有的 JavaScript 函数或库（需开启权限）

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 执行代码 | JavaScript | — | 必填。支持 `{{变量名}}` 引用上游变量 |
| 输出变量 | 文本 | — | 存储返回值的变量名。不填则丢弃返回值 |

## 常见问题

### 代码中无法访问 document / window

**现象**：代码中写了 `document.querySelector(...)` 但报错 undefined。

**原因**：未开启用户脚本权限，代码运行在沙箱环境中，无法访问页面 DOM。

**解决**：在 Chrome 扩展管理页面 `chrome://extensions` 中，找到九头虫 RPA → 开启「开发者模式」→ 开启「允许运行用户脚本」权限。开启后代码将在页面上下文中执行，即可访问 DOM。

### 返回值丢失或为 null

**现象**：代码执行成功但输出变量为 null。

**原因**：代码没有 return 语句，或返回值不是可序列化类型（如包含函数、DOM 元素引用、Symbol 等）。

**解决**：确保 return 的值是 JSON 兼容的基本类型（String、Number、Boolean、Object、Array）。
