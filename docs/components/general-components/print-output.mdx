# 打印输出

打印输出节点用于将指定内容输出到日志面板和浏览器控制台，是调试流程的主要手段。

## 概述

输出内容同时发送到两个地方：画布下方的日志面板和浏览器开发者工具 Console。内容为空时会报错——必须填写要输出的内容。

<img src="/img/打印输出.png" alt="打印输出节点参数配置面板" width="520" />

## 使用方法

拖入节点后，在「文本内容」中填写要输出的内容。支持 `{{变量名}}` 引用变量值，输出时变量会被替换为实际值。

典型调试用法：

- 在设置变量后加一个打印输出，确认变量值是否符合预期
- 在分支前打印判断变量的值，排查条件分支走错的原因
- 在关键节点前后加打印输出，追踪流程执行路径

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 文本内容 | 文本 | — | 必填。输出的内容，支持 `{{变量名}}` 变量引用 |

## 常见问题

### 高频循环中日志刷屏

**现象**：循环中大量输出导致关键信息被淹没。

**原因**：每次迭代都触发打印输出。

**解决**：在循环中配合「[条件分支](../flow-control/condition-branch)」，仅在出错或每 N 轮迭代时打印，减少无效输出。
