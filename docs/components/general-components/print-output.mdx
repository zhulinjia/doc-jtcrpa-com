# 打印输出

打印输出节点将内容输出到画布下方的日志面板，是调试流程的主要手段。

<img src="/img/打印输出.png" alt="打印输出节点参数配置面板" width="520" />

## 使用示例

**场景一：排查变量值是否符合预期**

在「[设置变量](./set-variable)」后接一个打印输出，确认赋值是否正确：

```
设置变量(score = 85) → 打印输出
```

文本内容填 `当前分数：{{score}}`，运行后在日志面板看到 `当前分数：85`，确认变量无误。

**场景二：排查条件分支为什么走错了**

在「[条件分支](../flow-control/condition-branch)」前各加一个打印输出，打印判断变量的值：

```
设置变量 → 打印输出(score = {{score}}) → 条件分支
```

日志面板直接显示变量值，不用猜分支走错的原因。

**场景三：追踪流程执行到哪一步**

在关键节点之间加标记性的打印输出：

```
打开网页 → 打印输出(页面已打开) → 等待元素 → 打印输出(列表已加载) → 数据采集
```

每一步完成都有日志输出，出错时一眼看出断在哪。

## 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 文本内容 | 文本 | — | 必填。支持 `{{变量名}}` 引用变量 |

## 常见问题

### 高频循环中日志刷屏

**现象**：循环中大量输出导致关键信息被淹没。

**解决**：在循环中配合「[条件分支](../flow-control/condition-branch)」，仅在出错或每隔 N 轮时打印，减少无效输出。
