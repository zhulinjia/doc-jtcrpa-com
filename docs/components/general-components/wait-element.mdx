# 等待元素

等待元素节点用于暂停流程执行，直到指定的 DOM 元素出现或消失后再继续。

## 概述

用于解决动态页面中元素延迟渲染导致的"找不到元素"问题。节点监听页面 DOM 变化，元素满足条件后立即继续。

支持两种监控类型：等待元素**出现**（最常见）和等待元素**消失**（如等待加载遮罩消失）。

## 使用方法

拖入节点后，配置以下三项：

### 目标元素

CSS 选择器，指定要监控的元素。支持 `{{变量名}}` 引用动态选择器，也支持「[CSS 选择器](/core-concepts/css-selectors)」中的所有标准语法和自定义伪类。

### 监控类型

- **元素出现**：页面中匹配到目标元素时继续。最常用的模式——等按钮渲染、等列表加载、等弹窗出现等
- **元素消失**：页面中不再匹配到目标元素时继续。适合等加载动画消失、等遮罩层关闭等场景

### 超时时间

最大等待秒数，**不填则默认 30 秒**。超时后节点抛错，流程中断。设置时考虑页面性能——网络慢或复杂页面适当调大。

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 目标元素 | CSS 选择器 | — | 必填。支持 `{{变量名}}` 引用 |
| 监控类型 | 下拉选择 | — | `appear` — 等待元素出现；`disappear` — 等待元素消失 |
| 超时时间（秒） | 数字 | 30 | 等待的最大时间，超时后抛错。不填即为默认 30 秒 |

## 常见问题

### 等待超时但元素明明在页面上

**现象**：元素已在页面显示，但节点仍然超时报错。

**原因**：选择器写错，或元素在 iframe 中而未切换上下文。

**解决**：在开发者工具 Console 中执行 `document.querySelector('你的选择器')` 验证；如在 iframe 中，先用「[切换 iframe](../browser-control/switch-iframe)」节点进入 iframe 上下文。
