# 延迟

延迟节点用于让流程暂停指定秒数后再继续执行。

## 概述

延迟是异步等待，不占用 CPU，不会阻塞浏览器渲染。适合以下场景：

- 等待页面动画或过渡效果完成
- 等待后端异步任务处理完毕再查询结果
- 控制请求频率，避免触发限流

## 使用方法

拖入节点后，设置「等待时长」（单位秒），执行即暂停对应时长后继续下游。

等待时长内部转换为毫秒（`秒数 × 1000`），最小值为 0（相当于不等待直接通过）。

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 等待时长（秒） | 数字 | — | 暂停的秒数，最小值为 0 |

## 常见问题

### 延迟结束后流程没有继续

**现象**：延迟结束但后续节点不执行。

**原因**：延迟节点到下游节点的连线断开，或下游节点本身在等待前置条件。

**解决**：检查延迟节点到下游的连线是否完整。
