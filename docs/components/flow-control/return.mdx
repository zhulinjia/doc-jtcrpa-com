# 返回

返回节点用于在子流程中结束执行并将数据返回给主流程。

## 概述

配合「[调用流程](./call-flow)」节点使用。子流程执行到返回节点时立即停止，返回值传给主流程中调用方配置的「输出变量」。仅在作为子流程被调用时生效——独立运行的流程中执行返回节点没有意义。

<img src="/img/返回.png" alt="返回节点参数配置面板" width="520" />

## 使用方法

在子流程末尾拖入返回节点，在 JSON 编辑器中填写返回值。

**支持 `{{变量名}}` 引用**——与调用流程的携带数据一样，变量会先替换再解析 JSON：

```json
{
  "status": "success",
  "amount": "{{price}}",
  "count": {{totalCount}},
  "data": {{resultList}}
}
```

变量中如果存储的是 JSON 字符串或数组，也可以直接嵌入，不加引号即可。

> **注意**：数字和布尔值不要加引号（`{{totalCount}}` 而非 `"{{totalCount}}"`），否则会被当作字符串。

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 返回值 | JSON5 | — | 返回给主流程的数据。支持 `{{变量名}}` 引用，先替换再解析。主流程通过「[调用流程](./call-flow)」的「输出变量」接收 |
