# 关闭网页

关闭网页节点用于关闭指定的浏览器标签页。

## 概述

在「关闭目标」下拉中选择要关闭的页面，执行即可。

<img src="/img/关闭页面.png" alt="关闭网页节点参数配置面板" width="520" />

## 使用方法

拖入节点后，在「关闭目标」下拉中选择要关闭的标签页即可。下拉列表中的选项来源如下：

| 选项 | 来源 |
|------|------|
| **当前页面** | 始终存在，指当前激活的标签页 |
| **初始页面** | 始终存在，指流程启动时已打开的标签页 |
| **其他页面名称** | 由流程中上游节点的「标签名」决定 |

### 页面名称从哪里来

下拉列表中除了「当前页面」和「初始页面」之外，其余选项的名称来自上游节点配置的**标签名**。具体来说：

- 上游「[打开网页](./open-page)」节点中填写的「**标签名**」，会成为下拉列表中的选项名称
- 部分事件交互节点（如「[点击事件](../event-interaction/click)」「[按下按键](../event-interaction/key-press)」）开启「**打开新标签页**」开关时，也需要填写标签名，同样会出现在下拉列表中

> 也就是说，你想关闭哪个页面，就取决于上游打开它时给它起的名字。

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 关闭目标 | 下拉选择 | — | 选择一个已打开的标签页进行关闭，选项由系统动态生成 |

## 常见问题

### 关闭标签页后流程中断

**现象**：关闭页面后后续节点报错。

**原因**：后续节点需要在已关闭的页面中执行。

**解决**：关闭前先使用「[切换页面](./switch-page)」节点将执行上下文转移到其他标签页。
