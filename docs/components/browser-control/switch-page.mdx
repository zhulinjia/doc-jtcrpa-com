# 切换页面

切换页面节点用于将浏览器焦点切换到另一个已打开的标签页。

## 概述

在「目标页面」下拉中选择要切换到的标签页，执行即可。

<img src="/img/切换页面.png" alt="切换页面节点参数配置面板" width="520" />

## 使用方法

拖入节点后，在「目标页面」下拉中选择要切换到的标签页。下拉列表中的选项来源如下：

| 选项 | 来源 |
|------|------|
| **当前页面** | 始终存在，指当前激活的标签页 |
| **初始页面** | 始终存在，指流程启动时已打开的标签页 |
| **其他页面名称** | 由流程中上游节点的「标签名」决定 |

### 页面名称从哪里来

下拉列表中除了「当前页面」和「初始页面」之外，其余选项的名称来自上游节点配置的**标签名**。具体来说：

- 上游「[打开网页](./open-page)」节点中填写的「**标签名**」，会成为下拉列表中的选项名称
- 部分事件交互节点（如「[点击事件](../event-interaction/click)」「[按下按键](../event-interaction/key-press)」）开启「**打开新标签页**」开关时，也需要填写标签名，同样会出现在下拉列表中

> 也就是说，你想切换到哪个页面，就取决于上游打开它时给它起的名字。

开启「**激活标签**」可在切换后自动将标签页聚焦到前台。

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 目标页面 | 下拉选择 | — | 选择一个已打开的标签页进行切换，选项由系统动态生成 |
| 激活标签 | 开关 | false | 切换到目标后是否自动聚焦 |

## 常见问题

### 下拉框中找不到想切换的页面

**现象**：目标标签页已打开，但下拉框中没有对应选项。

**原因**：该标签页不是由当前流程中的上游节点创建的。

**解决**：确保目标页面是通过本流程中的「[打开网页](./open-page)」节点或配置了「打开新标签页」的节点创建的。如果是手动打开的标签页，无法通过此节点切换。
