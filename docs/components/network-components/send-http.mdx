# 发送 HTTP 请求

发送 HTTP 请求节点用于直接向服务器发送 HTTP 请求，绕过页面 UI 与后端通信。

## 概述

相比通过页面 UI 触发请求更直接——不需要等待页面渲染、不依赖 DOM 元素。接口地址和各项参数均支持 `{{变量名}}` 引用。

<img src="/img/发送HTTP.png" alt="发送 HTTP 请求节点参数配置面板" width="520" />

## 使用方法

拖入节点后，按以下标签页分别配置：

### 基础配置

- **请求方法**：GET、POST、PUT、DELETE
- **接口地址**：必填，支持 `{{变量名}}` 引用。如 `https://api.example.com/data`

### 请求头

自定义请求头，JSON 键值对格式。会自动合并 `Content-Type`。

```json
{"Authorization": "Bearer {{token}}", "Accept": "*/*"}
```

### 查询参数

URL 查询参数，JSON 键值对格式。会自动拼接到接口地址末尾。

```json
{"page": "1", "size": "20"}
```

### 请求体

仅非 GET 方法可用。需选择 `Content-Type`（`application/json` / `text/plain` / `multipart/form-data` / `application/x-www-form-urlencoded`），请求体为 JSON 格式。

### 响应

- **响应类型**：`json`（默认）或 `text`
- **输出变量**：填写后响应数据存入该变量，供下游引用

:::tip[调试技巧]
调试 HTTP 请求时，推荐使用 [webhook.site](https://webhook.site/) ——打开即生成一个唯一 URL，把请求发过去即可实时查看请求方法、请求头、请求体等完整信息，无需自建后端。
:::

## 参数参考

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 请求方法 | 下拉选择 | GET | `GET` / `POST` / `PUT` / `DELETE` |
| 接口地址 | 文本 | — | 必填。支持 `{{变量名}}` 引用 |
| 请求头 | JSON | — | 自定义请求头，JSON 键值对 |
| 查询参数 | JSON | — | URL 查询参数，JSON 键值对，支持 `{{变量名}}` |
| Content-Type | 下拉选择 | — | `application/json` / `text/plain` / `multipart/form-data` / `application/x-www-form-urlencoded` |
| 请求体 | JSON | — | 请求体内容，仅非 GET 方法可用 |
| 响应类型 | 下拉选择 | json | `json` / `text` |
| 输出变量 | 文本 | — | 存储响应数据的变量名 |

## 常见问题

### 请求返回 401 或 403

**现象**：发送请求后收到未授权响应。

**原因**：接口需要特定的认证 Token 或 Cookie。

**解决**：在「请求头」中添加 `Authorization` 头，如 `{"Authorization": "Bearer {{token}}"}`。
