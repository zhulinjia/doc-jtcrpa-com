# 发送 HTTP 请求

发送 HTTP 请求节点直接向服务器发送请求，绕过页面 UI 与后端通信。比通过页面点击触发更快，也不依赖 DOM 元素。

<img src="/img/发送HTTP.png" alt="发送 HTTP 请求节点参数配置面板" width="520" />

## 使用示例

**场景：调用接口获取数据，处理后存入变量**

```
设置变量(token) → 发送 HTTP → 打印输出(响应结果)
```

- 请求方法：`GET`
- 接口地址：`https://api.example.com/orders?page=1`
- 请求头：`{"Authorization": "Bearer {{token}}"}`
- 响应类型：`json`
- 输出变量：`orders`

请求成功后，`{{orders}}` 就是接口返回的 JSON 数据，下游节点可以直接用 `{{orders.list}}`、`{{orders.total}}` 引用字段。

**场景：POST 提交数据**

```
发送 HTTP(POST, 请求体为 {{formData}}) → 打印输出
```

:::tip[调试技巧]
调试时推荐用 [webhook.site](https://webhook.site/)——打开即生成唯一 URL，把请求发过去即可实时查看完整的请求方法、请求头、请求体，无需自建后端。
:::

## 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 请求方法 | 下拉选择 | GET | `GET` / `POST` / `PUT` / `DELETE` |
| 接口地址 | 文本 | — | 必填。支持 `{{变量名}}` 引用 |
| 请求头 | JSON | — | 键值对格式，如 `{"Authorization": "Bearer {{token}}"}` |
| 查询参数 | JSON | — | URL 参数，自动拼接到地址末尾 |
| Content-Type | 下拉选择 | — | `application/json` / `text/plain` / `multipart/form-data` / `application/x-www-form-urlencoded` |
| 请求体 | JSON | — | 仅非 GET 方法可用，支持 `{{变量名}}` |
| 响应类型 | 下拉选择 | json | `json` / `text` |
| 输出变量 | 文本 | — | 存储响应数据的变量名 |

## 常见问题

### 请求返回 401 或 403

**原因**：接口需要认证 Token。

**解决**：在「请求头」中添加 `{"Authorization": "Bearer {{token}}"}`，Token 可通过「[设置变量](../general-components/set-variable)」提前配置。
